## `app_assets.dart` — принцип

**Усі шляхи до асетів — тільки тут.** Ніяких рядків у коді напряму.

```dart
AppAssets.invIconAll(isLightTheme)      // хелпер: dark/light-theme-all.png
AppAssets.donatButton / donatButtonActive
// Закоментований резерв:
// characterSex / characterRace / characterClass / armorySlots
```

|---|---|---|---|
| `true` | Світла (пергамент) | `wing_angel` | `dark-theme-*.png` |

## Верстка інвентаря

- `ItemCategory` enum: `potion, armor, weapon, useful, quest, accessory`
- `CategoryIcon` widget (замінив старий `CategoryGlyph`) — PNG з автовибором теми
- `iconPath(bool isLightTheme)` — extension-метод на `ItemCategory`
- У фільтрах використовується `CategoryIcon`, у тайлах списку — Material `Icon` (для кольору стану "екіпіровано")

---

## Кнопка донату (settings_dialog.dart)

`_DonatButton` — `StatefulWidget`:
- Звичайний стан: `AppAssets.donatButton`
- При натисканні: `AppAssets.donatButtonActive`
- Підпис: `tr(context, 'donateButton')` — перекладається
- Розміщена внизу діалогу налаштувань

## Vercel Analytics

Вже інтегровано у `web/index.html`:
```html
<script defer src="/_vercel/insights/script.js"></script>
```
**Не** використовувати `npm i @vercel/analytics` — це Flutter, не React.  
Щоб активувати: **Vercel Dashboard → проект → Analytics → Enable**.

## Конвенції

- Шляхи асетів → тільки через `AppAssets`
- Рядки UI → тільки через `tr(context, key)`
- `context.watch<AppState>()` у `build`, `context.read<AppState>()` поза `build`
- Нові переклади → додавати у всі 3 мови одразу (`uk`, `ru`, `en`)
- `CategoryIcon` замість Material Icons для фільтрів категорій інвентаря
